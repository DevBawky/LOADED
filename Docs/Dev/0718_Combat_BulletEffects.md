## AI-008: Custom Bullet Effects

### Basic Information

* Date: 260718
* Author: Yoon
* AI Tool: Codex
* Model: GPT-5
* Related Feature: Bullet / Critical / Debuff / Push / Position Swap

### Problem

기존 탄환은 대미지와 관통처럼 고정된 수치만 가질 수 있어, 독·기절·표식·밀치기·위치 교환·흡혈·약화 같은 효과를 탄환마다 조합하기 어려웠다.

효과별 발동 확률과 스택 수를 독립적으로 설정하고, 관통 및 크리티컬과 함께 처리될 때의 실행 순서를 명확히 정의할 필요가 있었다.

### Why AI Was Used

여러 효과가 한 탄환에 중첩될 때 발생하는 실행 순서, 처치된 적 처리, 관통 대상별 확률 판정과 같은 조합별 예외를 빠르게 정리하기 위해 AI를 사용했다.

기존 디버프, 직접 밀치기, 피해 계산 구조를 재사용하면서 ScriptableObject 직렬화 호환성을 유지하는 구현 범위를 함께 검토했다.

### Main Instructions

탄환마다 아래 효과를 원하는 수만큼 조합할 수 있는 시스템을 구현해주세요.

* 효과 종류는 `Poison`, `Stun`, `Mark`, `Knockback`, `PositionSwap`, `LifeSteal`, `Weakness`입니다.
* 각 효과는 별도의 발동 확률을 가지며, 같은 효과를 여러 번 등록할 수 있어야 합니다.
* 독·기절·표식·약화는 스택 수를, 밀치기는 이동 거리를 설정할 수 있어야 합니다.
* 관통 탄환은 적중한 각 대상에게 효과를 독립적으로 판정해야 합니다.
* 탄환마다 크리티컬 피해 배율을 설정하고, 플레이어에는 공통 크리티컬 확률을 추가해주세요.
* 크리티컬은 한 발마다 한 번 판정하고 해당 탄환의 모든 관통 대상이 결과를 공유하게 해주세요.
* 흡혈은 표시 대미지가 아닌 실제로 감소한 체력만큼 회복하고 최대 체력을 넘지 않게 해주세요.
* 탄환 밀치기는 플레이어의 기존 직접 밀치기 경로·충돌 규칙을 재사용하되 플레이어 위치와 밀치기 쿨다운은 변경하지 마세요.
* 위치 교환과 밀치기 애니메이션이 끝난 다음 후속 효과나 다음 탄환이 실행되어야 합니다.
* 기존 탄환 에셋의 enum 직렬화 값과 Inspector 참조를 유지해주세요.
* 구현 후 기존 탄환 에셋 마이그레이션 예시와 검증 항목을 정리해주세요.

### Output Summary

`BulletData`에 `BulletEffectData` 배열과 탄환별 `Critical Damage Multiplier`를 추가했다. 효과 데이터는 타입, 발동 확률, 디버프 스택 수, 밀치기 거리를 가진다.

Enum 직렬화 값은 다음 순서로 고정했다.

```text
Poison       = 0
Stun         = 1
Mark         = 2
Knockback    = 3
PositionSwap = 4
LifeSteal    = 5
Weakness     = 6
```

한 발의 처리 순서는 `적중 대상 확정 → 탄환 단위 크리티컬 판정 → 직접 피해 → Effects 배열 순서대로 효과 적용`이다. 직접 피해로 처치된 대상에는 흡혈만 허용하고 다른 상태·공간 효과는 건너뛴다.

주요 규칙은 다음과 같다.

* 표식은 새로 표식을 부여한 탄환이 아니라 이후 직접 공격부터 1.5배 피해를 적용한다.
* 약화는 직접 공격 피해를 30% 감소시키며 독과 밀치기 충돌 피해에는 적용하지 않는다.
* 흡혈은 크리티컬, 약화, 표식과 남은 체력을 모두 반영한 실제 감소 체력만 회복한다.
* 밀치기와 위치 교환은 코루틴 완료를 기다려 효과 배열 순서를 보장한다.
* 같은 효과가 여러 개면 각 항목을 독립적으로 판정하고 성공한 스택이나 회복을 누적한다.

기존 에셋은 `Ghost`의 턴 비소비, `Pierce`의 관통 설정을 유지했고, `Stun`, `Venom`, `Power`에는 기존 의도에 맞는 효과 항목을 연결했다.

### Validation

* 검증 방법:

  * 0%와 100% 효과 발동 경계값 코드 확인
  * 한 발의 관통 대상이 동일한 크리티컬 결과를 공유하는지 호출 흐름 확인
  * 처치 공격에서도 흡혈이 실제 감소 체력을 사용하는지 확인
  * 공간 효과 코루틴이 끝난 뒤 다음 효과가 진행되는지 확인
  * `dotnet build Assembly-CSharp.csproj --no-restore`로 C# 컴파일 확인

* 테스트 결과:

  * 확률 판정은 0%에서 항상 실패하고 100%에서 항상 성공하도록 구현되어 있다.
  * 디버프 스택과 공간 효과 설정은 효과 항목별로 독립 적용된다.
  * 현재 전체 C# 컴파일 결과는 경고 0개, 오류 0개다.
  * 실제 애니메이션 타이밍과 각 효과 조합의 Play Mode 확인은 수동 검증 항목으로 남겼다.

* 발견한 문제:

  * enum 순서를 변경하면 기존 에셋의 효과 타입이 다른 값으로 역직렬화될 수 있어 명시적인 정수 값을 유지해야 했다.
  * 직접 피해 후 사망한 대상을 일반 효과가 다시 조작하지 않도록 효과별 생존 조건을 구분해야 했다.

### Human Modifications

사용자가 직접 수정한 코드는 없다. 기존 프로젝트의 디버프 처리, 플레이어 밀치기와 EnemyData 구조를 기준으로 Codex가 통합했다.

효과별 전용 스프라이트와 최종 밸런스 값은 아트 및 밸런스 작업에서 별도로 연결해야 한다.

### Final Result

탄환 하나에 여러 효과를 원하는 순서로 등록하고 각 효과의 확률, 스택과 이동 거리를 개별 설정할 수 있게 되었다.

크리티컬, 관통, 표식, 약화, 흡혈과 공간 이동 효과가 하나의 발사 흐름에서 정의된 순서로 동작하며 기존 탄환 에셋의 직렬화 호환성도 유지했다.

### Lessons Learned

복합 효과 시스템은 효과 목록만 추가하는 것으로 끝나지 않고 직접 피해와 사망 판정, 공간 이동, 다음 탄환 실행 시점을 먼저 정의해야 안정적으로 확장할 수 있다.

enum을 ScriptableObject에 저장할 때는 기존 정수 값을 고정하고 새 항목을 마지막에 추가해야 에셋 마이그레이션 위험을 줄일 수 있다.
