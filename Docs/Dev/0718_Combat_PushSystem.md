### Problem

플레이어의 양옆 타일이 적으로 막히면 기존 이동 시스템에서는 어느 방향으로도 이동할 수 없다. 인접한 적을 밀어 공간을 만들 수 있는 밀치기 기능이 필요하다.

밀쳐진 적은 단순히 한 칸만 이동하는 것이 아니라 밀친 방향의 보드 끝까지 이동해야 한다. 이동 경로에 다른 적이 있다면 해당 적의 바로 앞 타일에 정지하고, 충돌한 두 적에게 각자의 최대 체력에 비례한 피해를 적용해야 한다.

밀치기 애니메이션과 피해 처리가 끝나기 전에 적 턴이 시작되면 적의 위치 판정과 행동 순서가 어긋날 수 있으므로, 전체 밀치기 처리가 끝난 이후에만 플레이어 턴을 완료해야 한다. 또한 밀치기를 연속으로 사용할 수 없도록 턴 기반 재사용 대기시간이 필요하다.

### Why AI Was Used

플레이어 이동, 보드 타일 좌표, 적 점유 상태, 적 이동 애니메이션, 피해 처리, 턴 완료 이벤트를 하나의 흐름으로 연결해야 했다.

특히 다음 예외 상황을 함께 처리하기 위해 AI를 사용했다.

* 밀친 방향에 다른 적이 없는 경우
* 다른 적이 한 칸 이상 떨어져 있는 경우
* 두 적이 바로 붙어 있어 밀쳐질 공간이 없는 경우
* 밀쳐진 적이 충돌 피해로 처치되는 경우
* 밀치기 도중 적 턴이 시작되는 문제
* 플레이어의 기절 턴을 포함한 쿨다운 계산

### Main Instructions

플레이어가 바라보는 방향의 바로 옆 타일에 적이 있으면 일반 이동 대신 밀치기를 실행한다. 플레이어 뒤쪽의 적은 이동 입력으로 밀칠 수 없다.

* 밀쳐진 적은 밀친 방향으로 계속 이동한다.
* 밀쳐진 적은 타일마다 뛰지 않고 최종 목적지까지 한 번에 날아간다.
* 이동 경로에 다른 적이 없다면 보드의 마지막 타일까지 이동한다.
* 다른 적을 만나면 해당 적의 바로 앞 타일에 정지한다.
* 충돌 시에는 상대 적의 모퉁이 접점에 부딪힌 뒤 정지 타일로 떨어지는 연출을 재생한다.
* 충돌한 두 적은 각자 최대 체력의 10% 피해를 받는다.
* 피해 비율과 타일 한 칸당 이동 시간은 Inspector에서 변경할 수 있어야 한다.
* 플레이어 충돌 연출, 적 밀치기 이동과 충돌 피해가 모두 끝난 뒤 적 턴을 시작한다.
* 밀치기는 사용 후 기본 3턴이 지나야 다시 사용할 수 있다.

### Output Summary

#### PlayerMove

`Assets/Scripts/Player/PlayerMove.cs`에 밀치기 설정과 실행 흐름을 추가했다.

플레이어가 이동할 인접 타일을 검사했을 때 적이 없으면 기존 이동을 실행한다. 적이 있다면 입력 방향과 플레이어가 바라보는 방향을 비교한다. 두 방향이 같고 밀치기 쿨다운이 끝난 상태에서만 `PushRoutine`을 시작한다. 뒤쪽 적이거나 쿨다운 중이라면 해당 방향으로 이동하거나 밀치지 않는다.

`TryBuildPushPath`는 밀쳐질 적의 현재 타일부터 밀친 방향으로 타일을 순회한다. 빈 타일은 이동 경로에 추가하고, 다른 적을 발견하면 순회를 종료한다. 따라서 경로의 마지막 위치는 다른 적의 바로 앞 타일이 되며, 다른 적이 없다면 보드 끝 타일이 된다.

`PushRoutine`의 처리 순서는 다음과 같다.

1. 밀쳐질 적과 이동 경로를 확정한다.
2. 플레이어 루트는 고정하고 Avatar만 적에게 부딪혔다가 원위치로 돌아오는 연출을 시작한다.
3. 이동 거리로 총 비행 시간을 계산하고 적을 최종 목적지까지 한 번에 날린다.
4. 충돌 대상이 있으면 양쪽 적에게 충돌 피해를 적용한다.
5. 다음 밀치기 사용 가능 턴을 계산한다.
6. `CompleteTurn`을 호출하여 적 턴을 시작한다.

충돌 대상이 있으면 밀쳐진 적은 정지 타일과 충돌 대상 사이의 접점까지 날아간다. 접점은 정지 타일보다 조금 높은 위치로 계산되며, 접점에 닿은 뒤 정지 타일로 짧게 가속 낙하한다. 낙하가 끝난 다음 충돌 피해를 적용한다.

두 적이 바로 붙어 있으면 최종 정지 타일은 밀쳐진 적의 현재 타일이다. 이 경우에도 밀쳐진 적은 상대 적의 모퉁이에 부딪혔다가 원래 타일로 떨어지는 연출을 재생한다. 밀치기 결과나 처치 여부와 관계없이 플레이어 루트는 원래 타일을 유지한다.

### Bullet Knockback Reuse

탄환의 `Knockback` 효과도 같은 경로 계산, 비행, 충돌 접점, 낙하, 최대 체력 비례 충돌 피해를 사용한다. 탄환 효과의 `Knockback Distance`만큼만 타일을 검사하며 그 범위 안에서 보드 끝 또는 다른 적을 만나면 정지한다.

탄환 밀치기는 플레이어가 직접 몸으로 미는 행동이 아니므로 Player Avatar의 충돌 왕복 연출을 재생하지 않는다. 플레이어 위치, 직접 밀치기 쿨다운과 `Next Push Available Turn`도 변경하지 않으며, 탄환 효과 모션이 끝난 뒤에만 다음 효과나 다음 탄환으로 진행한다.

#### WaveManager

`Assets/Scripts/Manager/WaveManager.cs`에 `TryGetEnemyAtTile`을 추가했다.

이 메서드는 지정한 타일에 살아 있는 적이 있는지 검사하고 해당 `EnemyController`를 반환한다. 기존 `IsTileOccupied`도 같은 조회 메서드를 사용하도록 변경해 적 점유 판정을 한 곳에서 처리한다.

#### EnemyController

`Assets/Scripts/Enemy/EnemyController.cs`에 다음 기능을 추가했다.

* `MaxHealth`: 적 데이터에 설정된 최대 체력을 반환한다.
* `ApplyCollisionDamage`: 표식 공격 배율을 적용하지 않는 직접 충돌 피해를 처리한다.
* `FlyTo`: 보드 끝과 같이 충돌 대상이 없는 목적지까지 한 번에 비행한다.
* `FlyIntoCollision`: 충돌 접점까지 비행한 뒤 최종 정지 타일로 낙하한다.

충돌 피해는 양쪽 적의 최대 체력을 각각 기준으로 계산한다. 최대 체력이 서로 다른 적끼리 충돌하면 각 적이 받는 피해도 서로 다를 수 있다.

#### ActorMotion

`Assets/Scripts/Common/ActorMotion.cs`에 단일 비행을 처리하는 `FlyTo`와 충돌 후 낙하까지 처리하는 `FlyIntoCollision`을 추가했다.

기존 일반 이동은 기존 `Move Duration`을 그대로 사용한다. 밀치기의 총 비행 시간은 이동 타일 수와 `Push Tile Duration`을 곱해서 계산하지만, 실제 모션은 타일마다 끊지 않고 시작점에서 목적지까지 하나의 포물선으로 재생한다.

충돌 모션은 접점까지의 단일 비행과 정지 타일로 돌아오는 낙하로 나뉜다. 낙하의 세로 진행에는 제곱 가속을 적용해 접점에서 잠시 박혔다가 아래로 떨어지는 느낌을 만든다.

### Cooldown Rule

밀치기 쿨다운은 `PlayerMove.TurnCount`를 기준으로 계산한다.

기본값이 3일 때 밀치기를 사용한 직후 남은 쿨다운은 3이다. 이후 플레이어 턴이 한 번 끝날 때마다 1씩 감소하며, 3번의 추가 턴이 모두 끝난 다음 행동부터 다시 밀치기를 사용할 수 있다.

이동, 회전, 대기, 재장전, 사격뿐 아니라 기절로 건너뛴 플레이어 턴도 턴 수에 포함된다.

외부 UI에서는 다음 공개 API를 사용할 수 있다.

* `PlayerMove.CanPush`: 현재 밀치기 사용 가능 여부
* `PlayerMove.RemainingPushCooldownTurns`: 남은 재사용 대기 턴
* `PlayerMove.PushCooldownChanged`: 플레이어 턴 종료 후 남은 쿨다운을 전달하는 이벤트

### Inspector Setup

`Assets/Prefabs/Player/Player.prefab`을 선택하고 `PlayerMove` 컴포넌트의 `Push` 섹션을 설정한다.

* `Push Collision Damage Ratio`
  * 충돌한 각 적이 자신의 최대 체력에서 잃는 비율이다.
  * 기본값 `0.1`은 최대 체력의 10%를 의미한다.
  * 실제 정수 피해는 올림 처리하며, 비율이 0보다 크면 최소 1의 피해를 준다.
* `Push Tile Duration`
  * 밀쳐진 적의 이동 거리 한 칸당 비행 시간이다.
  * 기본값은 `0.1`초다.
  * 최종 비행 시간은 이 값과 이동 거리의 곱이지만 애니메이션은 한 번에 재생된다.
  * `0`으로 설정하면 목적지 또는 충돌 접점까지 즉시 이동한다.
* `Push Collision Impact Ratio`
  * 정지 타일에서 충돌 대상 중심까지의 구간 중 실제 접점의 위치 비율이다.
  * 기본값 `0.5`는 두 타일 중심의 중간 지점을 사용한다.
* `Push Collision Impact Height`
  * 충돌 접점을 정지 위치보다 얼마나 높게 잡을지 설정한다.
  * 기본값은 `0.2`다.
* `Push Collision Settle Duration`
  * 접점에 부딪힌 후 정지 타일로 떨어지는 시간이다.
  * 기본값은 `0.12`초다.
* `Push Cooldown Turns`
  * 밀치기 사용 후 다시 사용할 때까지 필요한 추가 플레이어 턴 수다.
  * 기본값은 `3`이다.
  * `0`으로 설정하면 다음 플레이어 행동부터 다시 사용할 수 있다.
* `Next Push Available Turn`
  * 런타임 계산 상태다.
  * 현재 턴 수를 기준으로 자동 갱신되므로 직접 설정하지 않는다.

추가 컴포넌트나 UI 버튼 연결은 필요하지 않다. 기존 `MoveLeft`, `MoveRight`, `MoveForward`가 인접 적을 자동으로 감지한다. 단, 적을 밀치는 입력 방향이 플레이어의 현재 `transform.localScale.x` 방향과 같아야 한다. 빈 타일을 향한 일반 이동은 기존처럼 앞뒤 방향 모두 가능하다.

### Collision Examples

`P`는 플레이어, `A`는 밀쳐질 적, `B`는 충돌 대상 적, `_`는 빈 타일을 의미한다.

#### 충돌 대상이 없는 경우

```text
Before: P A _ _ _
After : P _ _ _ A
```

적 A는 보드 끝까지 이동하고 플레이어는 원래 타일에 남는다. 충돌 피해는 없다.

#### 떨어진 적과 충돌하는 경우

```text
Before: P A _ _ B
After : P _ _ A B
```

적 A는 적 B의 바로 앞에 정지하고 두 적 모두 각자 최대 체력의 설정된 비율만큼 피해를 받는다.

#### 적이 바로 붙어 있는 경우

```text
Before: P A B _ _
After : P A B _ _
```

적 A가 이동할 빈 타일이 없으므로 위치는 변하지 않는다. A와 B는 충돌 피해를 받고, A가 살아 있다면 플레이어도 이동하지 않는다.

### Play Mode Test Checklist

1. 플레이어 바로 옆에 적 한 명만 배치하고 해당 방향으로 이동한다.
2. 플레이어가 적의 반대 방향을 바라보는 상태에서는 밀치기가 실행되지 않는지 확인한다.
3. 적 방향으로 회전한 뒤 다시 이동해 밀치기가 실행되는지 확인한다.
4. 적이 타일마다 뛰지 않고 보드 끝까지 한 번에 날아가며 플레이어 루트는 원래 타일에 남는지 확인한다.
5. 밀칠 방향의 중간 타일에 다른 적을 배치한다.
6. 밀쳐진 적이 상대 적의 접점에 부딪혔다가 바로 앞 타일로 떨어지는지 확인한다.
7. 두 적의 최대 체력이 다를 때 각자 최대 체력의 10% 피해를 받는지 확인한다.
8. 두 적을 연속된 타일에 붙여 놓고 밀치기를 사용한다.
9. 밀쳐진 적이 상대 적의 접점까지 갔다가 원래 타일로 떨어지고, 플레이어는 제자리에 남는지 확인한다.
10. 밀치기 애니메이션 중 적 행동이 시작되지 않는지 확인한다.
11. 밀치기 사용 직후 남은 쿨다운이 3인지 확인한다.
12. 플레이어 턴을 세 번 더 완료한 뒤 밀치기를 다시 사용할 수 있는지 확인한다.

### Human Modifications

사용자가 직접 수정한 밀치기 코드는 없다.

기존 Scene 변경 내용과 전투 설정은 유지하고 `Player` 프리팹의 `PlayerMove` 설정에 밀치기 기본값만 추가했다.

### Final Result

플레이어는 자신이 바라보는 방향의 바로 옆 적만 기존 이동 입력으로 밀칠 수 있다. 뒤쪽 적은 밀칠 수 없지만, 빈 타일을 향한 기존 후방 이동은 그대로 사용할 수 있다.

밀쳐진 적은 다른 적과 충돌하거나 보드 끝에 도착할 때까지 타일별로 끊기지 않는 하나의 비행 모션으로 이동한다. 다른 적과 충돌하면 접점에 박힌 뒤 최종 정지 타일로 떨어지는 연출을 재생한다.

다른 적과 충돌하면 두 적은 각자 최대 체력에 비례한 직접 피해를 받으며, 밀치기와 플레이어 Avatar 복귀가 모두 끝난 이후에만 적들이 행동한다.

밀치기 피해 비율, 타일별 이동 시간, 재사용 대기 턴은 모두 `PlayerMove` Inspector에서 조정할 수 있다.

### Lessons Learned

턴제 전투에서 강제 이동은 위치만 변경하는 기능이 아니다. 이동 경로 확정, 점유 판정, 애니메이션, 충돌 피해, 처치 이벤트, 플레이어 위치 변경, 턴 완료 순서를 하나의 작업 단위로 처리해야 한다.

특히 턴 완료 이벤트를 애니메이션보다 먼저 호출하면 적 AI가 이동 중인 좌표를 기준으로 행동할 수 있다. 따라서 강제 이동과 피해 처리가 완전히 끝난 뒤 턴을 넘기는 순서가 중요하다.

재사용 대기시간을 프레임이나 시간 대신 플레이어의 완료된 턴 수로 계산하면 이동, 공격, 대기, 기절 등 서로 다른 행동에서도 일관된 규칙을 유지할 수 있다.
